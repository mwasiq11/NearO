# Bug Fixes Summary - January 26, 2026

## Issues Fixed

### ✅ 1. Service Creation 400 Bad Request
**Problem**: Service creation was failing with 400 Bad Request error

**Root Cause**: 
- Validation schema required latitude/longitude together (`.and('latitude', 'longitude')`)
- User was sending latitude/longitude but values might not be in valid range
- Availability field validation too strict

**Solution**: Modified `backend/src/utils/validationSchemas.js`
```javascript
// Changed from strict validation to flexible
availability: Joi.string().min(1).max(1000).required().trim(),
latitude: Joi.number().optional().allow(null),
longitude: Joi.number().optional().allow(null),
neighborhood: Joi.string().max(255).optional().allow('').trim(),
city: Joi.string().max(255).optional().allow('').trim()
// Removed .and('latitude', 'longitude') requirement
```

**Status**: ✅ FIXED

---

### ✅ 2. Discovery Trending Endpoint 500 Error
**Problem**: `/discover/trending` endpoint returning 500 Internal Server Error

**Error Message**: `ER_WRONG_ARGUMENTS - Incorrect arguments to mysqld_stmt_execute`

**Root Cause**: 
- `parseInt(limit)` was passing NaN when limit was undefined
- MySQL execute expects valid number for LIMIT parameter

**Solution**: Modified `backend/src/controllers/discovery.js`
```javascript
// Before:
params.push(parseInt(limit));

// After:
const limitNum = parseInt(limit, 10) || 20;
params.push(limitNum);
```

**Status**: ✅ FIXED

---

### ✅ 3. Email Verification Link Wrong URL
**Problem**: Verification email sent localhost:3000 URL (backend) instead of localhost:8080 (frontend)

**Impact**: Users couldn't click verification link since frontend runs on port 8080

**Solution**: Modified `backend/src/services/emailService.js`
```javascript
// Verification Email
const verificationUrl = `${process.env.FRONTEND_URL || 'http://localhost:8080'}/verify-email?token=${verificationToken}`;

// Password Reset Email
const resetUrl = `${process.env.FRONTEND_URL || 'http://localhost:8080'}/reset-password?token=${resetToken}`;
```

**Configuration**: Add to `.env` file:
```
FRONTEND_URL=http://localhost:8080
```

**Status**: ✅ FIXED

---

### ✅ 4. Audit Logging Missing for Signup/Login
**Problem**: User signup and login not appearing in `audit_logs` table in database

**Root Cause**: 
- Old implementation used direct SQL INSERT without proper helper
- Missing IP address and user-agent tracking
- Inconsistent metadata format

**Solution**: Modified `backend/src/controllers/authController.js`

#### Changes Made:

**1. Added Import**:
```javascript
import { logAudit, buildRequestContext } from '../audit/logger.js';
```

**2. Updated Register Function**:
```javascript
// Log signup in audit trail
const ctx = buildRequestContext(req);
await logAudit({
  actorId: id,
  actionType: 'user_signup',
  entityType: 'user',
  entityId: id,
  metadata: { name, email, role: 'user' },
  ipAddress: ctx.ipAddress,
  userAgent: ctx.userAgent
});
```

**3. Updated User Login Function**:
```javascript
// Log audit trail
const ctx = buildRequestContext(req);
await logAudit({
  actorId: user.id,
  actionType: 'user_login',
  entityType: 'user',
  entityId: user.id,
  metadata: { name: user.name, email: user.email, role: user.role },
  ipAddress: ctx.ipAddress,
  userAgent: ctx.userAgent
});
```

**4. Updated Moderator Login Function**:
```javascript
// Log audit trail
const ctx = buildRequestContext(req);
await logAudit({
  actorId: user.id,
  actionType: 'moderator_login',
  entityType: 'moderator',
  entityId: user.id,
  metadata: { name: user.name, email: user.email },
  ipAddress: ctx.ipAddress,
  userAgent: ctx.userAgent
});
```

**5. Updated Admin Login Function**:
```javascript
// Log audit trail
const ctx = buildRequestContext(req);
await logAudit({
  actorId: user.id,
  actionType: 'admin_login',
  entityType: 'admin',
  entityId: user.id,
  metadata: { name: user.name, email: user.email },
  ipAddress: ctx.ipAddress,
  userAgent: ctx.userAgent
});
```

**Audit Log Now Includes**:
- ✅ Actor ID (user who performed action)
- ✅ Action type (user_signup, user_login, moderator_login, admin_login)
- ✅ Entity type and ID
- ✅ Metadata with name, email, role
- ✅ IP address
- ✅ User agent (browser info)
- ✅ Timestamp (auto-generated by database)

**Status**: ✅ FIXED
